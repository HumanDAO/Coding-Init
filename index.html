Instructions

Using the Terminal as the code operation's manager

Installing Node.JS:

  1. clone / copy the files to local project directory.
  2. npm init
  3. npm install
  4. node server.js

Pulling/Pushing to Github Repo:

  1. mkdir <name>
  2. git clone <url>
  3. git add --all
  4. git commit -am " <message> "
  5. git push

Creating your Github url http:username.github.io/<reponame>/<projectname>/ from the gh-pages branch via Terminal:

  1. git branch
  2. git checkout -b gh-pages
  3. git merge master <------- if you updated master
  4. git push origin gh-pages
  5. git checkout master <------- go back to master, where you make your updates to code

Operating Heroku Live Server:

  1. git remote remove heroku \ If the domain/url/app name changes
  2. git remote add heroku (url) \ Add automatically generated URL domain name
  3. git remote -v \Get a list view with heroku
  4. heroku create \ Create a random Heroku server
  5. git remote -v \ This will register a remote git endpoint which should now show Github (origin) and Heroku
  6. heroku ps:scale web=1 \ To spin-up some resources for your app
  7. git push heroku master \ Finally push your code repo up to Heroku (yes it can be pushed to Github OR Heroku!)
  8. Anytime you push to the same heroku server, you only need to repeat step 6.
